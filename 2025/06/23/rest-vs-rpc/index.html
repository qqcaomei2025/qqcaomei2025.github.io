RESTful API 和RPC 在微服务开发中经常被提及，面试中也总是被问到。

起源和设计原则

REST

REST 是由Roy Fielding 2000在他的博士论文里面提出的。REST 是REpresentational State Transfer 的首字母组成的，是分布式超媒体系统的一种架构设计风格。REST并非具体的协议或者标准，它提供了一组设计原则和约束条件，用于指导分布式系统的构建。

REST有6个准则：

  1. 统一的接口
    4个约束条件保证了这一准则。识别资源，通过表述操作资源，自描述消息，超媒体驱动
  2. 客户端，服务端
    客户端，服务端分离
  3. 无状态
  4. 可缓存
    响应必须明确标识是否可缓存
  5. 分层
    客户端无需感知是否直接连接终端服务器
  6. 按需代码

RPC

RPC 代表远程过程调用，是一种用于分布式系统之间通信的古老而传统的协议。核心思想是像调用本地函数一样调用远程服务
